## 2. 배포(업데이트) 방법

사이트는 GitHub에 파일을 올리면 자동 반영됩니다.

**방법 A — GitHub 웹 업로드(가장 쉬움)**
1. https://github.com/whdcks498/haegwang 접속(구글 로그인).
2. 우측 `Add file ▾ → Upload files`, 또는 주소창에 `https://github.com/whdcks498/haegwang/upload/main`.
3. 수정한 파일을 끌어다 놓기 → 하단 **Commit changes** 클릭.
4. 1~2분 뒤 사이트 반영. (캐시 때문에 안 보이면 주소 뒤에 `?v=2` 붙여 새로고침)

**방법 B — git (명령어 사용 시)**
- `git add . && git commit -m "내용" && git push`

> 새 파일을 폴더 안(예: `.github/workflows/`)에 만들 땐 웹에서 `Add file → Create new file` 후 파일명에 `폴더명/파일명` 입력.

## 3. 채용공고 관리 (★ 클라우드로 전환됨 — 2026-06-15)

**가장 큰 변경:** 예전엔 공고가 올린 컴퓨터에서만 보였지만(localStorage), 이제 **Firebase 클라우드**에 저장되어 **모든 사람·모든 기기에서 동일하게 보입니다.**

- Firebase 프로젝트: **haegwang-cargo** (콘솔: https://console.firebase.google.com → 구글 whdcks498 로그인)
- 공고 데이터 위치: Firestore → 컬렉션 `jobs`
- **공고 등록:** 사이트에서 기업회원(승인) 또는 관리자 로그인 후 "공고 등록" → 자동으로 클라우드 저장 → 즉시 모두에게 노출.
- **허위·스팸 공고 삭제(점검 필터링):**
  1. Firebase 콘솔 → Firestore Database → 데이터 → `jobs`
  2. 지울 공고 문서 클릭 → 우측 **⋮ → 문서 삭제 → 삭제 시작**
  3. 사이트에 즉시 반영됨. (주기적으로 점검해 부적절한 공고 제거)
- 보안규칙: 누구나 읽기·등록 가능, 수정·삭제는 콘솔에서만(위 방법).

**아직 안 된 것(다음 단계):**
- 기업회원 **가입·로그인은 아직 기기별 저장**이라, 다른 컴퓨터에서 같은 계정 로그인은 안 됩니다(공고 '등록'은 가입한 그 PC에서). → 회원도 클라우드 전환 예정.
- 공고 **수정·삭제의 사이트 내 버튼**은 클라우드 공고엔 아직 미연동(현재는 콘솔에서 삭제).
