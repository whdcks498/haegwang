**최초 작성 2026-06-15 · 집 코워크가 작업 때마다 갱신**

> 이 문서는 우리 팀의 "업무 매뉴얼 + 로드맵 + 반복작업 일정"입니다. **회사 코워크(부사수)는 이 문서를 읽고 메모리에 저장**한 뒤 그대로 따릅니다. 대표님은 결정만, 실행은 두 코워크가 분담합니다.

|---|---|---|
| **대표 (1위)** | 방향 결정·최종 승인·계정/키 확인 | 박종찬 대표 |
| **집 코워크 (2위, 실무총괄)** | 기획·개발·문서·인수인계 갱신·집에서 원격 작업 | 해광AI비서 |

**원칙:** 부사수(회사 코워크)는 집 코워크의 인수인계를 따른다. 충돌 시 대표 결정 우선.

## 2. 공통 업무 규칙
1. **착수 전에만 승인** → 이후 끝까지 자동 실행.
2. **배포 전 항상 GitHub 동기화** (`git fetch` + `git reset --hard origin/main`). 집·회사 양쪽 작업이라 필수.
3. 배포 = GitHub 웹 업로드(`/upload/main`) 또는 git push → 1~2분 후 사이트 반영.
4. "안 된다"로 끝내지 말 것 — 화면 캡처·웹검색·원격으로 끝까지.
5. 소통: 대표님께 간결한 한국어. 큰 변경 후 인수인계 문서 갱신.

## 3. 코워크 작업 시작 루틴 (매번)

1. GitHub 최신 받기 (`git reset --hard origin/main`)
2. 이 폴더의 인수인계 문서 읽기 (아래 4번 목록)
3. 핵심을 메모리에 저장 / 갱신
4. 대표님께 오늘 할 일 확인 후 진행
